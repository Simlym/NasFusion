# 索引设计策略

## 1. 主要索引类别

### 1.1 唯一索引（UNIQUE INDEX）
- 确保数据唯一性
- 支持快速查找
- 防止重复数据

### 1.2 复合索引（COMPOSITE INDEX）
- 支持多字段查询
- 优化常用查询模式
- 减少回表操作

### 1.3 条件索引（PARTIAL INDEX）
- 针对特定条件的数据
- 减少索引大小
- 提高查询效率

### 1.4 JSON索引（GIN INDEX）
- 支持JSON字段查询
- 全文搜索支持
- 复杂数据结构查询

---

## 2. 查询优化策略

### 2.1 常用查询模式
1. **按时间范围查询**：`INDEX(field, created_at DESC)`
2. **按状态过滤**：`INDEX(status, priority DESC)`
3. **按用户查询**：`INDEX(user_id, created_at DESC)`
4. **全文搜索**：`FULLTEXT INDEX` on text fields

### 2.2 分页查询优化
- 使用覆盖索引避免回表
- 合理的索引顺序支持分页
- 避免大偏移量查询

### 2.3 统计查询优化
- 预计算统计字段
- 使用物化视图
- 合理的聚合索引
