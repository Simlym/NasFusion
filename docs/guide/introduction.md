# 项目介绍

**NasFusion** 是一个专为 NAS 用户设计的 PT 媒体资源全生命周期管理系统。从资源发现、自动下载，到文件整理、媒体库集成，一站式搞定。

## 为什么是 NasFusion？

传统工具（如 Radarr/Sonarr）的工作流程是：**先在 TMDB 搜索你想看的** → 再去 PT 站点找资源。这种方式有明显缺陷：

- 每次搜索都要实时访问 PT 站点，速度慢、封号风险高
- 无法感知站点的促销资源（免费、2X 上传等）
- 对 PT 站点的资源结构不够友好

NasFusion 反其道而行之：

> **PT 优先** — 先把 PT 站点的资源同步到本地数据库，再在本地闪电搜索、智能匹配。

## 核心工作流

```
PT 站点资源
    │
    ▼
资源同步到本地数据库（定时 / 手动）
    │
    ├──► 搜索浏览（毫秒级响应）
    │
    ├──► 订阅匹配（新资源 → 自动下载）
    │         │
    │         ▼
    │    qBittorrent 下载
    │         │
    │         ▼
    │    文件整理（硬链接 / 软链接）
    │         │
    │         ▼
    │    元数据刮削（TMDB / 豆瓣）
    │         │
    │         ▼
    │    刷新 Jellyfin / Emby / Plex
    │
    └──► 通知推送（Telegram / 邮件）
```

## 功能全览

### 已实现功能

| 模块 | 核心功能 | 状态 |
|------|----------|------|
| 🌐 **PT 站点管理** | 多站点配置、健康检查、代理支持 | ✅ |
| 🔄 **资源同步** | 定时同步、去重、促销识别、季集解析 | ✅ |
| 🔍 **资源搜索** | 关键词搜索、多维度筛选、即时下载 | ✅ |
| 🎯 **资源识别** | 自动识别类型、TMDB/豆瓣元数据关联 | ✅ |
| 📺 **订阅追剧** | 季度订阅、质量优先级、全自动追更 | ✅ |
| 📁 **文件整理** | 硬链接整理、规则自定义、目录结构化 | ✅ |
| 🎬 **媒体服务器** | Jellyfin/Emby/Plex 集成、观看历史同步 | ✅ |

| 🔔 **通知系统** | Telegram、邮件通知、事件驱动 | ✅ |

## 技术栈

| 层次 | 技术选型 |
|------|----------|
| **后端** | FastAPI + SQLAlchemy 2.0 (Async) |
| **数据库** | PostgreSQL（生产）/ SQLite（开发） |
| **前端** | Vue 3 + TypeScript + Element Plus |
| **任务调度** | APScheduler + asyncio |
| **AI 集成** | OpenAI API（智能推荐） |
| **容器化** | Docker + Docker Compose |
