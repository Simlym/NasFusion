# 下载管理

NasFusion 通过适配器与 qBittorrent 集成，统一管理所有下载任务，并在下载完成后自动触发后续的文件整理流程。

## 支持的下载器

| 下载器 | 状态 | 说明 |
|--------|------|------|
| qBittorrent | ✅ 已支持 | 主要支持，功能完整 |
| Transmission | 🚧 开发中 | 计划支持 |
| Download Station | 🚧 计划中 | 群晖 NAS 内置下载器 |

## 配置 qBittorrent

### 1. 开启 qBittorrent Web UI

在 qBittorrent 中：
1. 进入 **工具 → 选项 → Web UI**
2. 勾选「启用 Web 用户界面」
3. 设置端口（默认 8080）和用户名/密码
4. 点击保存

<!-- 图：qBittorrent Web UI 设置截图 -->
<!-- ![qBittorrent 配置](/images/download/qb-webui-config.png) -->

::: tip 图片预留位置
*qBittorrent 选项页截图：高亮显示 Web UI 设置部分*
:::

### 2. 在 NasFusion 中配置

进入 **设置 → 下载器 → 添加下载器**：

| 字段 | 说明 | 示例 |
|------|------|------|
| 下载器类型 | 选择 qBittorrent | - |
| 名称 | 自定义名称 | "本地 qB" |
| 地址 | Web UI 地址（含端口） | `http://192.168.1.100:8080` |
| 用户名 | Web UI 登录用户名 | `admin` |
| 密码 | Web UI 登录密码 | - |
| 默认分类 | 下载任务的 qB 分类 | `NasFusion` |
| 默认保存路径 | 下载文件保存目录 | `/downloads` |

配置完成后点击 **测试连接** 验证。

### 任务状态

| 状态 | 说明 |
|------|------|
| ⏳ 等待中 | 已添加到 qBittorrent，等待开始 |
| ⬇️ 下载中 | 正在下载 |
| 🌱 做种中 | 下载完成，正在做种 |
| ✅ 已完成 | 下载并整理完成 |
| ❌ 出错 | 下载失败 |
| ⏸ 已暂停 | 手动暂停 |

### 任务操作

在任务列表中可以：

- **暂停 / 继续**：控制单个任务
- **重新下载**：对失败任务重试
- **删除任务**：可选择是否同时删除文件
- **查看详情**：文件列表、Tracker 状态、速度历史

## 下载监控

系统每 **5 分钟**从 qBittorrent 同步一次任务状态，检测到下载完成时自动触发：

```
下载完成（qBittorrent 状态变为"做种中"）
    │
    ▼
NasFusion 下载监控任务检测到
    │
    ▼
标记任务为"已完成"
    │
    ├──► 发布事件：download_completed
    │
    ├──► 文件整理服务：创建硬链接、整理到媒体库
    │
    ├──► 元数据刮削：获取海报、简介等
    │
    ├──► 通知媒体服务器刷新
    │
    └──► 推送通知（Telegram / 邮件）
```

## 批量下载

在资源搜索结果中可以批量选择资源下载：

1. 勾选多条资源
2. 点击 **批量下载**
3. 选择下载器和保存路径
4. 确认后批量推送到 qBittorrent

## 常见问题

**Q：添加下载任务后 qBittorrent 没有出现新任务？**

检查：
1. NasFusion 中下载器地址是否能访问（通过测试连接确认）
2. Docker 环境下地址是否使用了宿主机 IP 而非 `localhost`
3. qBittorrent Web UI 是否已开启

**Q：下载完成后文件没有整理到媒体库？**

检查：
1. **设置 → 目录管理** 中的媒体库路径是否正确
2. 下载目录和媒体库是否在同一文件系统（硬链接模式需要）
3. 查看任务日志中是否有错误信息
