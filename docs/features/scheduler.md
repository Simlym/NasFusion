# 任务调度

NasFusion 内置基于 APScheduler 的任务调度系统，自动化管理所有后台任务，让系统真正做到「无人值守」运行。

进入 **任务调度** 查看所有任务的运行状态：

## 内置任务

| 任务名称 | 默认频率 | 说明 |
|----------|----------|------|
| **PT 资源同步** | 每 6 小时 | 同步所有已启用站点的最新资源 |
| **站点健康检查** | 每 1 小时 | 验证各站点连通性和 API Key 有效性 |
| **订阅匹配** | 资源同步后触发 | 新资源与活跃订阅进行匹配 |
| **下载任务监控** | 每 5 分钟 | 同步 qBittorrent 下载状态 |
| **文件整理** | 下载完成后触发 | 整理下载完的文件到媒体库 |
| **媒体库同步** | 每天 02:00 | 与媒体服务器同步媒体信息 |
| **观看历史同步** | 每天 03:00 | 从媒体服务器拉取最新观看记录 |
| **存储空间检查** | 每 6 小时 | 检查磁盘剩余空间，低于阈值告警 |

## 手动触发任务

在任务列表中，点击任意任务右侧的 **立即执行** 按钮，可以跳过等待时间手动触发：

<!-- 图：手动触发按钮截图 -->
<!-- ![手动触发](/images/scheduler/manual-trigger.png) -->

::: tip 图片预留位置
*任务行截图：高亮显示"立即执行"按钮位置*
:::

常用场景：
- 刚添加新 PT 站点后，立即触发同步
- 修改订阅配置后，立即重新匹配
- 手动触发健康检查验证站点状态

## 修改执行频率

点击任务右侧的 **编辑** 按钮修改执行计划：

<!-- 图：频率配置编辑弹窗截图 -->
<!-- ![频率配置](/images/scheduler/frequency-config.png) -->

::: tip 图片预留位置
*频率编辑弹窗截图：Cron 表达式输入框（带可视化帮助）或简单的"每N小时"下拉选择*
:::

支持两种配置方式：

**简单模式**（适合大多数用户）：

| 选项 | 示例 |
|------|------|
| 每 N 分钟 | 每 5 分钟 |
| 每 N 小时 | 每 6 小时 |
| 每天 HH:MM | 每天 02:30 |

**Cron 模式**（灵活）：

```
# 格式：分 时 日 月 周
0 */6 * * *     # 每 6 小时的整点
30 2 * * *      # 每天凌晨 2:30
0 2 * * 1       # 每周一凌晨 2:00
```

## 启用 / 禁用任务

点击任务前的开关可以临时禁用某个任务：

- **禁用**：任务不再自动执行（但可以手动触发）
- 常见场景：PT 站点维护期间暂停同步任务

::: warning
禁用「订阅匹配」任务后，新资源不再自动与订阅匹配，需手动触发或重新启用。
:::

## 任务执行历史

点击任务名称进入详情页，查看历史执行记录：

<!-- 图：任务历史记录截图 -->
<!-- ![任务历史](/images/scheduler/task-history.png) -->

::: tip 图片预留位置
*任务历史页截图：时间轴列表，每条记录显示执行时间、耗时、执行结果（成功/失败）、处理数量*
:::

每次执行记录包含：

| 字段 | 说明 |
|------|------|
| 开始时间 | 任务开始执行的时间 |
| 结束时间 | 任务完成时间 |
| 耗时 | 执行总耗时 |
| 状态 | ✅ 成功 / ❌ 失败 / ⚠️ 部分成功 |
| 摘要 | 如"同步 234 条资源（新增 45 条）" |
| 错误信息 | 失败时的错误详情 |

```
[14:30:02] 获取第 1 页资源（50 条）
[14:30:03] 获取第 2 页资源（50 条）
...
[14:31:45] 同步完成：共 732 条（新增 48 条，更新 684 条）
[14:31:45] 触发订阅匹配...
[14:31:47] 订阅匹配完成：找到 3 条匹配，触发 3 个下载任务
```

::: tip
任务日志保留最近 30 天，超期自动清理。
:::

## 任务依赖关系

部分任务之间存在触发关系：

```
资源同步完成
    └──► 自动触发 → 订阅匹配

下载完成（监控检测到）
    └──► 自动触发 → 文件整理
    └──► 自动触发 → 元数据刮削
    └──► 自动触发 → 通知媒体服务器刷新
```

这些依赖关系由事件系统实现，确保处理顺序正确。
