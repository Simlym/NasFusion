# 通知系统

NasFusion 采用事件驱动的通知架构，在关键事件发生时（下载完成、站点异常等）即时推送通知，让你随时掌握系统动态。

## 支持的通知渠道

| 渠道 | 状态 | 适用场景 |
|------|------|----------|
| **Telegram** | ✅ 已支持 | 实时推送，支持富文本和按钮 |
| **邮件（SMTP）** | ✅ 已支持 | 适合不常用 Telegram 的用户 |
| **Webhook** | 🚧 开发中 | 接入自定义服务（如企业内网） |
| **企业微信** | 🚧 计划中 | 企业用户 |

## 配置 Telegram 通知

### 步骤 1：创建 Telegram Bot

1. 打开 Telegram，搜索 **@BotFather**
2. 发送 `/newbot`，按提示填写 Bot 名称和用户名
3. 获得 **Bot Token**（格式：`1234567890:ABCDEFabcdef...`）

### 步骤 2：获取 Chat ID

方法一（推荐）：
1. 打开 Telegram，搜索 **@userinfobot**
2. 发送任意消息，Bot 会回复你的 Chat ID

方法二（获取群组 ID）：
1. 将 Bot 添加到群组
2. 访问 `https://api.telegram.org/bot{TOKEN}/getUpdates`
3. 在返回的 JSON 中找到 `chat.id`（群组 ID 为负数）

进入 **设置 → 通知 → Telegram**：

## 配置邮件通知

进入 **设置 → 通知 → 邮件**：

| 字段 | 说明 | 示例 |
|------|------|------|
| SMTP 服务器 | 邮件服务器地址 | `smtp.gmail.com` |
| 端口 | SMTP 端口 | `587`（TLS）/ `465`（SSL） |
| 用户名 | 发件邮箱地址 | `yourname@gmail.com` |
| 密码 | 邮箱密码或应用专用密码 | - |
| 收件地址 | 通知发送到哪个邮箱 | `notify@gmail.com` |
| 启用 TLS | 推荐开启 | ✅ |

::: tip 常见 SMTP 配置

**Gmail**：
```
SMTP: smtp.gmail.com:587 (TLS)
需要开启"两步验证"并生成"应用专用密码"
```

**QQ 邮箱**：
```
SMTP: smtp.qq.com:587 (TLS)
密码使用授权码（在 QQ 邮箱设置中开启 SMTP 后获取）
```

**163 邮箱**：
```
SMTP: smtp.163.com:465 (SSL)
密码使用授权码
```
:::

## 通知事件

以下事件会触发通知推送，可以在设置中单独开关每种事件：

### 下载相关

| 事件 | 说明 | 重要性 |
|------|------|--------|
| ✅ 下载完成 | 某个资源下载并整理完毕 | ⭐⭐⭐ |
| ❌ 下载失败 | 任务下载出错 | ⭐⭐⭐ |
| 📺 订阅匹配 | 追剧订阅找到新集并开始下载 | ⭐⭐⭐ |
| ⏱ 下载超时 | 任务长时间未完成 | ⭐⭐ |

### 系统相关

| 事件 | 说明 | 重要性 |
|------|------|--------|
| ⚠️ 站点异常 | PT 站点连接失败（连续 3 次） | ⭐⭐⭐ |
| 💾 磁盘空间不足 | 剩余空间低于阈值 | ⭐⭐⭐ |
| 🔄 同步完成 | 资源同步任务完成摘要 | ⭐ |
| 🎬 媒体库更新 | 新媒体入库 | ⭐ |

## 通知设置

### 全局开关

在 **设置 → 通知** 中可以：
- 一键开启/关闭所有通知
- 针对每种渠道（Telegram/邮件）分别控制
- 针对每类事件分别控制

<!-- 图：通知设置页截图 -->
<!-- ![通知设置](/images/notification/settings-page.png) -->

::: tip 图片预留位置
*通知设置页截图：左侧渠道配置，右侧事件开关列表（勾选框控制每类事件是否通知）*
:::

### 免打扰时段

可以设置免打扰时段（如深夜），在此时段内通知会被静默，下次活跃时间再发送。

```
免打扰时段：23:00 - 08:00
```

### 通知频率限制

为避免频繁推送，系统有以下限制：
- 同一事件的通知最多 **1 次/分钟**
- 批量下载完成会合并为一条摘要通知

## 测试通知

配置完成后，点击 **发送测试通知** 按钮：

- Telegram：会立即收到一条测试消息
- 邮件：会在几秒内收到一封测试邮件

如果测试通知没有收到，请检查：
1. Bot Token 和 Chat ID 是否填写正确
2. Bot 是否已被封禁（尝试给 Bot 发消息看是否有响应）
3. 是否需要配置代理（在 Telegram 服务受限的地区）
