# 媒体库管理

NasFusion 在下载完成后自动完成文件整理、元数据刮削，并通知媒体服务器刷新媒体库，实现「下载即可观看」的无缝体验。

## 整体流程

```
qBittorrent 下载完成
        │
        ▼
文件整理（硬链接 / 软链接 / 复制）
        │
        ├─ 解析文件名 → 确定目标路径
        │
        └─ 创建目录结构 → 链接文件
                │
                ▼
        元数据刮削（TMDB / 豆瓣）
                │
                ├─ 下载海报、背景图
                └─ 生成 .nfo 文件
                        │
                        ▼
                通知媒体服务器刷新
                        │
                        ├─ Jellyfin 刷新媒体库
                        ├─ Emby 刷新媒体库
                        └─ Plex 刷新媒体库
```

### Jellyfin 配置

| 字段 | 说明 | 示例 |
|------|------|------|
| 服务器地址 | 包含端口 | `http://192.168.1.100:8096` |
| API Key | 在 Jellyfin 控制台生成 | `abc123...` |
| 用户名 | 用于同步观看历史 | `admin` |

**获取 Jellyfin API Key**：
1. 打开 Jellyfin 管理控制台
2. 进入 **控制台 → API 密钥 → 新增**
3. 填写应用名称（如 "NasFusion"），复制生成的 Key

### Emby 配置

配置方式与 Jellyfin 相同，字段一致。

### Plex 配置

| 字段 | 说明 |
|------|------|
| 服务器地址 | `http://192.168.1.100:32400` |
| Plex Token | 从 Plex 账户获取 |

**获取 Plex Token**：
1. 登录 Plex Web
2. 打开任意媒体的 **...菜单 → 获取信息**
3. 在地址栏 URL 中找到 `X-Plex-Token=xxx` 参数

## 文件整理规则

### 整理模式

| 模式 | 说明 | 推荐场景 |
|------|------|----------|
| **硬链接** ⭐ | 不占用额外空间，做种不受影响 | 下载目录和媒体库在同一磁盘 |
| **软链接** | 跨磁盘时可用 | 下载和媒体库在不同磁盘 |
| **复制** | 完整复制文件 | 需要独立副本（占用双倍空间） |

::: tip 强烈推荐硬链接
使用硬链接时：
- 媒体库文件和 qBittorrent 下载目录指向同一数据块
- 删除媒体库中的文件不影响做种
- 不占用额外磁盘空间
- 需要下载目录和媒体库在同一文件系统
:::

### 目录规则配置

在 **设置 → 媒体整理** 中配置目录命名规则：

```
电影目标路径：{movie_dir}/{title} ({year})/
            示例：电影/复仇者联盟 (2012)/

剧集目标路径：{tv_dir}/{series_name}/Season {season:02d}/
            示例：电视剧/权力的游戏/Season 01/

文件名规则：{series_name} - S{season:02d}E{episode:02d} - {episode_title}
         示例：权力的游戏 - S01E01 - 凛冬将至
```

## 元数据刮削

整理完成后，系统自动从元数据源获取详细信息：

### 获取内容

| 内容 | 来源 |
|------|------|
| 海报图（poster.jpg） | TMDB |
| 背景图（fanart.jpg） | TMDB |
| 剧照（thumb.jpg） | TMDB |
| 简介 | TMDB（中文优先）/ 豆瓣 |
| 评分 | TMDB / 豆瓣双来源 |
| 演员表 | TMDB |
| 导演/编剧 | TMDB |
| 中文译名 | 豆瓣（更准确） |
| 类型标签 | TMDB |

### NFO 文件

刮削结果保存为 `.nfo` 文件，Jellyfin/Emby/Plex 可直接读取：

```xml
<?xml version="1.0" encoding="utf-8"?>
<movie>
  <title>复仇者联盟</title>
  <originaltitle>The Avengers</originaltitle>
  <year>2012</year>
  <rating>8.0</rating>
  <plot>当邪恶的洛基...</plot>
  <director>乔斯·韦登</director>
  <actor><name>小罗伯特·唐尼</name><role>托尼·斯塔克</role></actor>
  ...
</movie>
```

## 观看历史同步

NasFusion 定期从媒体服务器同步观看历史：

```
Jellyfin / Emby / Plex
         │
         │  API 拉取观看记录
         ▼
    NasFusion 数据库
         │
         ├──► AI 推荐系统（基于观看历史分析偏好）
         └──► 订阅完结检测（全季看完 → 归档订阅）
```

同步的观看数据包括：
- 播放时间和播放进度
- 是否看完（>90% 视为已看完）
- 评分（如果用户在媒体服务器内评分）

## 目录管理

在 **设置 → 目录管理** 中配置：

- **媒体库根目录**：整理后文件的存放位置
- **下载目录**：qBittorrent 下载目录（用于创建硬链接）
- **临时目录**：处理过程中的临时文件

::: warning 注意路径映射
如果使用 Docker 部署，容器内路径和宿主机路径可能不同，需要在 `docker-compose.yml` 中正确配置挂载点：

```yaml
volumes:
  - /your/media/path:/media         # 媒体库
  - /your/downloads/path:/downloads  # 下载目录
```

NasFusion 使用容器内路径（`/media`、`/downloads`），qBittorrent 使用宿主机路径。二者指向同一物理路径即可正常使用硬链接。
:::
