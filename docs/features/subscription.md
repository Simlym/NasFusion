# 订阅与自动下载

订阅系统是 NasFusion 的核心功能，实现「一次配置，全自动追剧」。为一部剧集创建订阅后，系统会自动监测新资源、匹配并触发下载，全程无需人工干预。

## 工作流程

```
创建订阅（指定剧集 + 季度 + 质量偏好）
         │
         ▼
资源同步（定时 / 手动触发）
         │
         ▼
订阅匹配（新资源 vs 活跃订阅）
         │
         ├──► 未匹配：等待下次同步
         │
         └──► 匹配成功
                  │
                  ├─ 质量打分排序
                  │
                  ├─ 已下载？→ 跳过
                  │
                  └─ 触发下载（推送到 qBittorrent）
                           │
                           ├──► 通知推送
                           └──► 状态更新
```

### 配置订阅参数

| 参数 | 说明 | 示例 |
|------|------|------|
| **季度** | 选择要追的季（支持多选） | 第 1 季、第 2 季 |
| **质量偏好** | 期望的视频质量 | 1080p / 4K / 最佳 |
| **字幕偏好** | 对字幕的要求 | 中文字幕 / 国配 / 无要求 |
| **编码偏好** | 视频编码格式 | H.265 优先 / H.264 / 无要求 |
| **大小限制** | 单集文件大小上限 | ≤ 10GB |
| **来源站点** | 从哪些站点匹配 | 全部 / 指定站点 |
| **下载路径** | 覆盖默认下载目录 | 可选，留空用默认路径 |

### 高级选项

展开「高级选项」可进一步配置：

| 参数 | 说明 |
|------|------|
| **做种数下限** | 只下载做种数 ≥ N 的资源 |
| **优先促销资源** | 有免费资源时优先选择 |
| **关键词过滤** | 标题必须/不能包含特定词 |
| **替换质量** | 已下载后若出现更好质量则替换 |

## 质量优先级

当同一集有多个资源可选时，系统按以下规则打分排序：

### 分辨率评分

```
4K HDR    → 100 分
4K        → 90 分
1080p REMUX → 85 分
1080p     → 80 分
720p      → 60 分
其他      → 40 分
```

### 同质量下的排序（更细粒度）

1. **促销优先**：免费资源优先（可关闭）
2. **做种数**：做种越多越靠前（资源稳定性）
3. **文件大小**：同质量下更小的文件优先
4. **上传时间**：更新的资源优先

### 字幕偏好

```
选择"中文字幕"时：
  标题含 "中字" / "CHT" / "CHS" → +20分

选择"国配"时：
  标题含 "国语" / "国配" → +30分
```

### 手动补全

若某集长期未找到资源，可手动操作：

1. 点击对应集数
2. 手动搜索资源
3. 点击 **强制下载** 跳过质量检查

## 防重复下载

系统通过以下机制避免重复下载：

- 追踪每集的 `download_hash`（种子哈希）
- 相同 hash 不重复添加到 qBittorrent
- 订阅更新（如从 720p 升级到 1080p）时，会先暂停旧任务再下载新资源

## 完结检测

系统会定期检测订阅的完结状态：

1. 从 TMDB 获取该剧集的状态（`status: ended`）
2. 确认全部集数都已下载
3. 自动将订阅状态改为「已完结」
4. 发送通知：「《XX》全季下载完成！」

## 使用场景示例

### 追当季新番

1. 新番开播前（或开播后），搜索剧集创建订阅
2. 设置偏好：1080p、中文字幕
3. 开启「优先促销资源」（新番常有免费）
4. 系统每次同步后自动匹配、下载，发 Telegram 通知

### 补全老剧全季

1. 搜索老剧，选择全部季度创建订阅
2. 触发手动同步
3. 系统一次性找到全季资源，批量下载
4. 全部下载完成后，自动归档

### 高清替换

已有 720p 的旧资源，想替换为 1080p：

1. 开启订阅的「替换质量」选项
2. 系统在同步到 1080p 资源时，自动暂停旧任务、开启新下载
